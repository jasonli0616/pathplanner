Create an auto builder that will create command groups that will handle path following and triggering events.

This auto builder will use RamseteCommand to follow paths.

Parameters
poseA function that supplies the robot pose - use one of the odometry classes to provide this.
resetPoseA consumer that accepts a Pose2d to reset robot odometry. This will typically be called once at the beginning of an auto.
controllerThe RAMSETE controller used to follow the trajectory.
kinematicsThe kinematics for the robot drivetrain.
feedforwardThe feedforward to use for the drive.
speedsSupplierA function that supplies the speeds of the left and right sides of the robot drive.
driveConstantsPIDConstants for each side of the drive train
outputOutput consumer that accepts left and right voltages
eventMapMap of event marker names to the commands that should run when reaching that marker.
driveRequirementsThe subsystems that the path following commands should require. Usually just a Drive subsystem.
useAllianceColorShould the path states be automatically transformed based on alliance color? In order for this to work properly, you MUST create your path on the blue side of the field.
